다음 자료를 바탕으로 “미적분Ⅰ 사진 문제풀이봇 MVP”를 구현해줘.

## 목표
학생이 문제를 직접 타이핑하지 않고 사진을 업로드하면, OpenRouter의 이미지 입력 가능 모델이 문제를 읽고, `mijeokbun1_ai_rag_cards_v0_1.jsonl`의 AI용 개념카드를 참고해서 단계별 풀이를 제공하는 웹앱을 만든다.

## 중요한 전제
- 학생용 개념노트는 교사가 나중에 별도로 업로드한다.
- 이 JSONL 파일은 학생에게 보여주는 개념노트가 아니라, AI 풀이 근거/RAG 검색용 자료다.
- OpenRouter API 키는 절대 프론트엔드에 노출하지 말고 서버 환경변수로만 사용한다.
- 모델명도 환경변수로 관리한다.

## 사용해야 할 파일
- `mijeokbun1_ai_rag_cards_v0_1.jsonl`
- `mijeokbun1_solver_system_prompt_v0_1.md`
- `CODEX_HANDOFF_mijeokbun1_photo_solver_v0_1.md`
- `mijeokbun1_test_cases_v0_1.jsonl`

## 필수 UX
1. 학생이 문제 사진을 업로드한다.
2. 사진 미리보기를 보여준다.
3. AI가 인식한 문제를 [문제 읽기]에 정리한다.
4. 관련 개념카드 1~3개를 검색한다.
5. 정해진 7개 섹션으로 풀이를 출력한다.
   - [문제 읽기]
   - [사용 개념]
   - [풀이 전략]
   - [단계별 풀이]
   - [정답]
   - [검산]
   - [비슷한 문제 팁]
6. 풀이에는 사용한 개념카드 ID를 표시한다.

## 구현 요구
- `retrieveRelevantCards(problemText, cards, topK)` 함수를 분리해서 만들어라.
- MVP에서는 키워드/문자열 검색으로 충분하다.
- 추후 벡터 검색으로 바꿀 수 있도록 구조화해라.
- 수식은 Markdown + LaTeX가 깨지지 않게 렌더링해라.
- 에러가 나면 학생에게 쉬운 문구로 안내하라.

## 완료 기준
- 사진 업로드가 가능하다.
- OpenRouter 호출이 서버에서만 일어난다.
- JSONL 개념카드 검색이 작동한다.
- 풀이 형식이 항상 7개 섹션으로 나온다.
- 테스트 케이스 일부를 직접 넣었을 때 관련 카드 ID가 제대로 잡힌다.
